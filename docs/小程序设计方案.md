# 九型芯之力 · 微信小程序设计方案

> 版本 v1.0 ｜ 2026-06 ｜ 基于现有「官网(website-react) + 管理后台(nx-backend)」延伸设计

---

## 一、背景与目标

### 1.1 现状盘点（已有资产）

| 模块 | 现状 | 可被小程序复用 |
|------|------|----------------|
| 官网 website-react | 首页 / 老师简介 / 九型三阶段 / 课程 / 课件 / 小游戏 / 企业课程 | 内容与品牌资产、九型测试逻辑 |
| 九型测试小游戏 | 18 题情境测试 + 三中心 + 副型 + 分享卡片 | **直接迁移并升级**（核心引流入口） |
| Go 后端 (nx-backend) | JWT 鉴权、PostgreSQL、站点配置、报名线索、统计、MiniMax 语音(TTS)、上传(OSS/本地) | **整套 API 复用**，只需新增微信侧接口 |
| 管理后台 (Vben) | 官网管理 / 客户管理(报名) / 系统管理(RBAC/品牌) | **直接扩展**小程序内容/用户/订单管理 |
| 已有公开接口 | `/api/public/site-config`、`/api/public/signups`、`/api/public/game-results`、`/api/public/site-visits` | 小程序内容与埋点可直接调用 |

> 关键结论：**后端无需重写**。小程序复用同一个 Go 服务 + 同一个 PostgreSQL + 同一个管理后台，只需新增「微信登录 + 小程序专属业务接口」。这是本方案最大的成本优势。

### 1.2 为什么做小程序

1. **更短的转化路径**：官网偏 PC/品牌展示，小程序是微信生态内的「即点即用」，适合九型测试这种社交传播型内容。
2. **沉淀用户资产**：官网是匿名访问，小程序通过微信登录沉淀真实用户(openid)，把「一次性测试者」变成「可触达的会员」。
3. **裂变能力**：微信分享、组队测试、海报、订阅消息是官网没有的增长杠杆。
4. **商业闭环**：深度解读报告、付费课程、1v1 咨询预约可在小程序内用微信支付直接成交。

### 1.3 业务目标（建议量化）

- 引流：九型测试 → 留资/关注（目标测试完成率 > 60%，留资率 > 15%）
- 转化：测试结果页 → 课程/咨询预约（目标预约转化 > 5%）
- 留存：会员体系 + 内容更新（目标 7 日留存 > 25%）
- 变现：深度报告付费 + 课程报名 + 企业课程线索

## 二、目标用户与核心场景

### 2.1 用户画像
- **个人成长用户**：对性格/情绪/关系感兴趣，被朋友分享的测试吸引进入。
- **关系困惑用户**：亲子、夫妻、职场关系，想通过九型理解他人。
- **企业/HR**：团队建设、领导力，关注企业课程。
- **老学员**：复训、查看课件、听语录、预约个案。

### 2.2 核心场景（用户旅程）
```
被分享/扫码 → 进入小程序 → 做九型测试（无需登录即可玩）
     → 看到基础结果 → 想看「深度报告 / 与TA的关系解读」
     → 微信一键登录，存档结果
     → 引导：免费内容（语录音频/三阶段科普） + 付费（深度报告/课程/预约）
     → 分享结果海报 → 拉新
     → 沉淀为会员，订阅消息触达复访
```

## 五、技术选型

### 5.1 前端框架：**推荐 uni-app（Vue3 + TS）**

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| **uni-app** | 一套代码可发**微信小程序 + H5 + 安卓/iOS App**；Vue3 与团队现有 React/Vue 经验接近；生态成熟、插件多 | 偶有平台差异需适配 | **推荐**：未来想上抖音小程序/App 时零额外成本 |
| Taro (React) | 与官网 React 技术栈一致 | 多端一致性略弱于 uni-app | 备选（若团队强 React） |
| 微信原生 | 性能/兼容最佳 | 只能微信单端，后续扩展要重写 | 不推荐（锁死单端） |

> 选 uni-app 的核心理由：官网用 React、后台用 Vue，团队两套都熟；uni-app 让**小程序未来可一键复用为 H5 和 App**，与现有官网形成「PC官网 + 移动小程序 + App」的统一矩阵。

### 5.2 后端：**复用现有 Go 服务**，不新增独立服务
- 同一个 `nx-backend/apps/server`，新增 `internal/wechat`（微信登录/支付/订阅消息）与 `internal/miniapp`（小程序业务接口）两个包。
- 同一个 PostgreSQL，新增小程序相关表（见第七节）。
- 同一个管理后台 Vben，新增「小程序管理」菜单组。

### 5.3 整体架构
```
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│ 微信小程序     │   │ 官网 website   │   │ 管理后台 Vben  │
│ (uni-app)     │   │ (React)       │   │ (Vue/Antd)    │
└──────┬───────┘   └──────┬───────┘   └──────┬───────┘
       │  HTTPS/JSON      │                  │
       └──────────────────┼──────────────────┘
                          ▼
              ┌─────────────────────────┐
              │   Go 后端 (nx-backend)    │
              │  auth / wechat / miniapp  │
              │  signup / voice / ...     │
              └────────────┬────────────┘
                           ▼
              ┌─────────────────────────┐
              │ PostgreSQL + 对象存储(OSS) │
              │ + MiniMax(TTS) + 微信API   │
              └─────────────────────────┘
```

## 七、后端改造点

### 7.1 新增接口清单（公开 = 无需登录，授权 = 需小程序 JWT）

```
微信登录
  POST /api/wx/login           授权  code → openid/session → 签发小程序JWT
  POST /api/wx/phone           授权  解密手机号（getPhoneNumber）
  GET  /api/wx/userinfo        授权  当前小程序用户资料
  PUT  /api/wx/userinfo        授权  更新昵称/头像

测试存档
  POST /api/miniapp/test-records         授权  保存一次测试结果
  GET  /api/miniapp/test-records         授权  我的测试历史
  POST /api/miniapp/relation-match       授权  关系合盘（两份结果对比）

内容
  GET  /api/public/contents              公开  课程/三阶段/图鉴列表（基于 site-config）
  GET  /api/public/voice-quotes          公开  语录音频列表（复用 voice 产物）

转化
  POST /api/miniapp/bookings             授权  提交咨询/课程预约（落 signups）
  GET  /api/miniapp/bookings             授权  我的预约
  POST /api/public/leads                 公开  企业课程留资（复用 signups）

会员与支付
  GET  /api/miniapp/products             公开  深度报告/课程商品列表
  POST /api/miniapp/orders               授权  下单 → 调起微信支付（统一下单）
  POST /api/wx/pay/notify                公开  微信支付回调（验签）
  GET  /api/miniapp/orders               授权  我的订单

触达
  POST /api/wx/subscribe-message         授权  发送订阅消息（预约提醒/报告就绪）
```

### 7.2 微信登录流程（核心）
```
小程序 wx.login() → code
   └─POST /api/wx/login {code}
        Go 后端 → 微信 code2session(appid+secret+code)
                → 拿到 openid / unionid / session_key
                → upsert wx_users（按 openid）
                → 生成本系统 JWT（payload 含 wx_user_id）
        ← { accessToken, userInfo }
小程序存 token，后续请求带 Authorization: Bearer <token>
```
> 复用现有 `internal/auth` 的 JWT 签发；只是登录来源从「用户名密码」扩展出「微信 openid」。

## 八、数据模型新增（PostgreSQL）

```sql
-- 小程序用户
wx_users(
  id, openid UNIQUE, unionid, nickname, avatar, phone,
  gender, main_type INT,             -- 最近一次测试主型
  member_level INT DEFAULT 0,        -- 0普通 1会员
  channel, scene,                    -- 来源渠道/场景值
  created_at, last_login_at
)

-- 测试记录（登录后存档）
test_records(
  id, wx_user_id, gender, result_type, second_type,
  scores JSONB, centers JSONB, created_at
)

-- 商品（深度报告/课程）
products(id, type, title, price_cents, cover, detail JSONB, status, sort)

-- 订单
orders(
  id, order_no UNIQUE, wx_user_id, product_id, amount_cents,
  status,            -- pending/paid/refunded/closed
  wx_transaction_id, paid_at, created_at
)

-- 预约（与现有 signups 打通：booking 关联 wx_user_id）
bookings(
  id, wx_user_id, kind,   -- consult/course/enterprise
  contact_name, phone, intent, preferred_time,
  status, signup_id,      -- 同步进后台客户管理
  created_at
)

-- 收藏 / 分享记录（可选，运营用）
favorites(id, wx_user_id, target_type, target_id, created_at)
share_logs(id, wx_user_id, scene, created_at)
```
> 现有 `signups` 表保持不变，`bookings` 通过 `signup_id` 写入一条线索，让**后台「客户管理」自动看到小程序来的预约**，运营无需切换系统。

## 九、核心模块详解

### 9.1 九型测试（引流核心）
- **免登录可玩**：降低门槛，做完才引导登录。
- **升级点**（相对官网）：
  - 登录后**存档 + 历史对比**（"3 个月前你是 7 号，现在更偏 5 号"）。
  - **关系合盘**：输入/邀请另一半也测，生成两型关系解读（差异化卖点）。
  - **深度报告**（付费 ¥9.9~19.9）：成长建议、亲密关系模式、职业倾向、压力管理——内容可由九型资料 + MiniMax AI 生成。
- 分享：结果海报（复用官网已做的 canvas 卡片逻辑，移植为小程序 canvas）。

### 9.2 内容中心
- 课程/三阶段/企业课程：直接读 `/api/public/site-config`，**后台改文案小程序即时更新**（已有运行时配置能力）。
- **老韩语录音频**：复用 `voice` 包的 MiniMax TTS 产物，做成「每日一条」音频卡片，增强复访与品牌人格。
- 九型图鉴：9 张型号详解卡，可分享单型。

### 9.3 转化与变现
- **预约**：选意向（个人成长/亲子/企业）+ 时段 + 留言 → 落 `signups`，后台客户管理跟进。
- **微信支付**：深度报告、付费课程、咨询定金。统一下单 + 回调验签 + 订单状态机。
- **会员**：付费/复购升级会员，解锁全部深度报告 + 专属内容。

### 9.4 AI 能力（差异化，复用 MiniMax）
- 已接入 MiniMax，可扩展为「**九型 AI 教练**」：基于用户主型做个性化问答/成长陪伴（付费会员权益），文本用 MiniMax LLM，语音用已有 TTS。

### 9.5 增长裂变
- 结果海报（带小程序码）→ 朋友圈/群。
- **组队测试**：邀请 N 位好友测试解锁团队报告（适合企业/家庭场景）。
- 订阅消息：预约提醒、报告生成、新语录上线。
- 分享有礼：邀请得报告券。

## 十二、合规与风控（小程序上线必须）

1. **小程序类目与资质**：性格测试/课程属「教育」或「工具」类目；涉及付费课程需相应资质，企业主体备案。
2. **个人信息保护**：登录授权、手机号获取需弹《隐私协议》；最小必要收集；提供注销账号入口（微信合规强制）。
3. **心理测评免责**：测试页明确「**仅供趣味参考，不构成心理诊断/医疗建议**」（官网已有此声明，沿用）。
4. **内容安全**：用户生成内容(留言/合盘邀请)接微信内容安全接口 `msgSecCheck/imgSecCheck`。
5. **支付合规**：商户号、退款流程、订单凭证留存。
6. **审核要点**：测试结果页不得诱导分享（按微信规范设计分享按钮）。
